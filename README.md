

**CryptoSafeManager** — локальный менеджер паролей с графическим интерфейсом, мастер-паролем, шифрованием базы данных, защищённым буфером обмена и журналом действий.

Проект предназначен для безопасного хранения логинов, паролей и заметок на компьютере пользователя без отправки данных на внешний сервер.

---

## Основные возможности

* создание защищённого хранилища паролей;
* вход по мастер-паролю;
* добавление, редактирование и удаление записей;
* поиск записей в хранилище;
* генерация надёжных паролей;
* хранение данных в зашифрованном виде;
* копирование логина и пароля в буфер обмена;
* автоматическая очистка буфера обмена;
* журнал действий пользователя;
* импорт и экспорт данных;
* блокировка хранилища;
* режим паники;
* готовый `.exe`-файл в папке `dist`.

---

## Используемые технологии

* **Python** — основной язык разработки;
* **Tkinter** — графический интерфейс;
* **SQLite** — локальная база данных;
* **Argon2id** — проверка мастер-пароля;
* **PBKDF2-HMAC-SHA256** — получение ключа шифрования;
* **AES-256-GCM** — шифрование записей;
* **PyInstaller** — сборка `.exe`;
* **pytest** — тестирование проекта.

---

## Как устроена защита

Мастер-пароль не хранится в открытом виде.

Общая схема работы:

```text
Мастер-пароль
      ↓
Argon2id
проверка правильности пароля
      ↓
PBKDF2-HMAC-SHA256
получение ключа шифрования
      ↓
AES-256-GCM
шифрование записей в базе данных
```

Пароли записей не хранятся обычным текстом. Каждая запись шифруется отдельно. Для каждой операции шифрования используется уникальный `nonce`.

### 1. Создать виртуальное окружение

```bash
python -m venv .venv
```

### 2. Активировать окружение

Для Windows:

```bash
.venv\Scripts\activate
```

### 3. Установить зависимости

```bash
pip install -r requirements.txt
```

Если на Windows возникают проблемы с зависимостями, можно использовать:

```bash
pip install -r requirements-win.txt
```

### 4. Запустить приложение

```bash
python run.py
```

## Работа с приложением

### Создание хранилища

При первом запуске пользователь создаёт мастер-пароль. Он нужен для входа в приложение и доступа к зашифрованным данным.

### Добавление записи

В хранилище можно добавить:

* название сервиса;
* логин;
* пароль;
* ссылку;
* заметки;
* теги.

### Копирование пароля

Пароль можно скопировать в буфер обмена. После копирования приложение автоматически очищает буфер через заданное время или при блокировке хранилища.

### Журнал действий

Приложение фиксирует важные события:

* вход в приложение;
* ошибки входа;
* добавление записей;
* изменение записей;
* удаление записей;
* импорт и экспорт;
* действия с буфером обмена.

В базе данных хранятся:

* мастер-пароль;

---

## Безопасность

В проекте используются следующие механизмы защиты:

* мастер-пароль не хранится в открытом виде;
* проверка мастер-пароля выполняется через Argon2id;
* ключ шифрования получается через PBKDF2-HMAC-SHA256;
* записи шифруются через AES-256-GCM;
* для каждой записи используется уникальный nonce;
* буфер обмена очищается автоматически;
* при блокировке очищаются чувствительные данные;
* режим паники быстро блокирует приложение и очищает временные данные;
* журнал действий помогает отслеживать важные операции.

---

## Ограничения

* проект является учебным приложением;
* системный буфер обмена не может быть полностью защищён от всех внешних программ;
* защита памяти в Python не является абсолютной;
* BitLocker не является частью приложения, но может использоваться дополнительно для защиты всего диска;
* облачная синхронизация и внешний сервер не используются.
