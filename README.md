# CryptoSafeManager


### Windows quick start

If `pip` fails with `Building wheel for argon2` / `Microsoft Visual C++ 14.0 is required`, run:

```bat
INSTALL_WINDOWS.bat
```

This installer skips the problematic old `argon2` package and starts the app with a PBKDF2 authentication fallback. For full Argon2id mode, install Python 3.12 x64 and run:

```bat
INSTALL_FULL_SECURITY_WINDOWS.bat
```

Безопасный менеджер паролей с открытым исходным кодом, построенный на принципах модульности и криптографической защиты. Проект развивается итерационно в рамках 8 спринтов, каждый из которых добавляет новый уровень функциональности и безопасности.

## Видение проекта

CryptoSafeManager призван предоставить пользователям надёжное и удобное средство для хранения конфиденциальной информации (пароли, заметки, данные кредитных карт) с акцентом на безопасность, расширяемость и современные стандарты шифрования. Архитектура проекта спроектирована таким образом, чтобы каждый следующий спринт мог быть реализован без кардинальной переработки уже написанного кода.

## Финальная версия

Эта сборка сохранена как **CryptoSafeManager** и включает обновленный интерфейс: верхний security header, левый dashboard, карточную рабочую область, современную toolbar-панель и улучшенную таблицу vault entries. Логика спринтов сохранена: мастер-пароль, AES-256-GCM, secure clipboard, audit chain, import/export, sharing, QR/key exchange, auto-lock, tray и panic mode.

Подробная карта реализации находится в `docs/sprint_implementation_map.md`, а изменения финальной версии — в `docs/release_notes.md`.

## Архитектура (MVC)

Проект следует паттерну Model-View-Controller (MVC), что обеспечивает чёткое разделение ответственности и упрощает тестирование и расширение.

**Основные компоненты:**

- **View (GUI)** — отвечает за отображение и взаимодействие с пользователем. Содержит главное окно, переиспользуемые виджеты и диалоги.
- **Controller (Core)** — содержит бизнес-логику: управление событиями, конфигурацией, состоянием приложения, криптографическими операциями.
- **Model (Database)** — обеспечивает работу с базой данных SQLite, создание таблиц, выполнение запросов, миграции и индексы.

## Установка и настройка

### Предварительные требования

- Python 3.9 или выше
- Git (для клонирования репозитория)

### Инструкция по установке

1. **Клонируйте репозиторий**

   ```bash
   git clone https://github.com/luckyof/CryptoSafe-Manager.git
   cd CryptoSafe-Manager
   ```

2. **Создайте виртуальное окружение**

   ```bash
   python -m venv venv
   ```

   Активируйте его:

   - **Windows:** `venv\Scripts\activate`
   - **Linux/macOS:** `source venv/bin/activate`

3. **Установите зависимости**

   Проект использует внешние зависимости для криптографии, проверки мастер-пароля, буфера обмена, QR/key exchange, тестирования и финальной сборки.

   ```bash
   pip install -r requirements.txt
   ```

   Основные зависимости:

   - `argon2-cffi`
   - `cryptography`
   - `pytest`
   - `pytest-cov`
   - `pyinstaller`
   - `pyperclip`
   - `qrcode`
   - `Pillow`
   - `pyzbar`
   - `opencv-python`
   - `pywin32` на Windows
   - `pyobjc-framework-Cocoa` на macOS

   Для Linux-буфера обмена может потребоваться один из системных backend-пакетов: `wl-clipboard`, `xclip` или `xsel`.

   На Windows можно использовать быстрый скрипт:

   ```bat
   INSTALL_WINDOWS.bat
   ```

   Если окружение уже создано, следующий запуск можно делать через:

   ```bat
   RUN_WINDOWS.bat
   ```

   Дополнительные heavy-зависимости для QR camera/image scanning находятся в `requirements-optional.txt`.

4. **Запустите приложение**

   ```bash
   python run.py
   ```

   Альтернативный запуск:

   ```bash
   python main.py
   ```

   При первом запуске откроется мастер первоначальной настройки, где нужно будет указать путь к базе данных и задать мастер-пароль.

После успешного запуска откроется главное окно программы. В Sprint 1 были реализованы базовые элементы интерфейса:

- Меню «Файл», «Правка», «Вид», «Справка».

- Строка состояния с индикатором статуса.

- Журнал аудита.

## Использование (2 спринт)

Во втором спринте были добавлены:

- Работа с мастер-паролем. Мастер-пароль никогда не хранится в явном виде. Для проверки используется соль + хеш Argon2id. При вводе неверного пароля доступ к данным блокируется.
- Защищённая память. Ключи шифрования хранятся в заблокированной памяти через `mlock` на Linux/macOS или `VirtualLock` на Windows. Это предотвращает выгрузку чувствительных данных в swap-файл.
- Атомарные транзакции. Операции смены пароля выполняются атомарно: либо полностью, либо с откатом. Это гарантирует целостность данных даже при сбое во время обновления.

## Использование (3 спринт)

В третьем спринте реализованы основные рабочие функции хранилища:

- Реальное шифрование записей через AES-256-GCM.
- CRUD-операции для записей хранилища.
- Мягкое удаление записей с переносом в `deleted_entries`.
- Генератор паролей с оценкой сложности.
- Поиск по полям `title`, `username`, `url`, `notes`, `category`.
- Field-specific фильтры вроде `category:"Work"`.
- Фильтрация по тегам.
- Фильтрация по диапазону дат (`created_at`, `updated_at`).
- Фильтрация по силе пароля.

Дополнительно:

- Ошибки доступа к записи не раскрывают существование конкретного `entry_id`.
- Реализован аудит ключевых операций с записями.
- Главное окно работает с реальными данными, а не с заглушками.

## Использование (4 спринт)

В четвёртом спринте добавлена защищённая работа с системным буфером обмена:

- новый модуль `src/core/clipboard/` с сервисом clipboard, платформенными адаптерами и монитором;
- кнопки и контекстное меню для копирования логина, пароля и данных записи;
- автоматическая очистка буфера обмена по таймеру;
- очистка при блокировке, выходе, закрытии приложения и замене содержимого;
- индикатор состояния буфера в строке статуса и подсветка записи, чьи данные находятся в буфере;
- masked preview содержимого с раскрытием через повторную аутентификацию;
- мониторинг внешнего изменения буфера и подозрительной активности;
- ускоренная очистка и возможность блокировки копирования после подозрительного события;
- настройки таймера, уведомлений, профилей безопасности и мониторинга;
- поддержка Windows/macOS/Linux backend-адаптеров и fallback через `pyperclip`;
- защита данных в памяти: XOR-маска, попытка `VirtualLock`/`mlock`, зануление после очистки;
- интеграция с `EntryManager`, audit log и future hooks для TOTP, secure sharing и panic mode.

Добавлены тесты Sprint 4 для clipboard-архитектуры, конфигурации, валидации, интеграции, производительности, security requirements и error handling.

## Состояние после Sprint 5

К состоянию пятого спринта проект дополнен полноценным журналом аудита и проверкой целостности. Реализованы подписанные записи аудита, цепочка хешей, автоматическая и ручная проверка, экспорт отчета проверки, безопасный экспорт журнала и импорт подписанного JSON для независимой проверки.

Основные возможности Sprint 5:

- журнал аудита фиксирует события входа, операций с хранилищем, буфером обмена, конфигурацией, TOTP, panic mode, импорта и экспорта;
- каждая audit-запись содержит `sequence_number`, `previous_hash`, `entry_hash`, подпись и публичный ключ;
- проверка целостности обнаруживает изменение, удаление, вставку и нарушение порядка записей;
- автоматическая периодическая проверка запускается таймером в `AuditLogger`;
- просмотр журнала доступен из GUI после аутентификации и работает постранично через `LIMIT/OFFSET`;
- из GUI можно запустить проверку, экспортировать отчет проверки и управлять расписаниями экспорта;
- экспорт поддерживает `json`, `csv`, `pdf` и `cef`, при необходимости с AES-256-GCM шифрованием;
- импорт подписанного JSON выполняет независимую проверку подписи и цепочки перед загрузкой;
- чувствительные поля не попадают в audit details в открытом виде.

## Быстрое использование основных функций

Этот раздел кратко описывает основные пользовательские сценарии финальной версии. Более подробная инструкция находится в `docs/user_guide.md`.

### Создание и разблокировка хранилища

1. Запустите приложение командой `python run.py` или через собранный `CryptoSafeManager.exe`.
2. При первом запуске мастер настройки попросит выбрать расположение базы данных и задать мастер-пароль.
3. При следующих запусках введите мастер-пароль для разблокировки хранилища.


### Работа с записями

- Кнопка **Добавить** создает новую запись с названием, логином, паролем, сайтом, категорией и тегами.
- Кнопка **Редактировать** открывает выбранную запись для изменения.
- Кнопка **Удалить** переносит запись в корзину.
- Двойной щелчок по записи открывает ее редактирование.
- Поиск и фильтры позволяют отбирать записи по названию, логину, URL, категории, тегам, дате и силе пароля.

### Генератор паролей

- В окне добавления или редактирования записи нажмите **Сгенерировать**.
- Настройте длину и набор символов.
- Нажмите кнопку использования сгенерированного пароля, чтобы автоматически подставить его в поле записи.

### Защищенный буфер обмена

- Выберите запись и нажмите **Копировать логин** или **Копировать пароль**.
- Данные помещаются в буфер обмена на ограниченное время.
- После таймера буфер автоматически очищается.
- При блокировке хранилища буфер также очищается.


### Импорт, экспорт и безопасный обмен

- **Экспорт** сохраняет выбранные данные в поддерживаемый формат, при необходимости с шифрованием.
- **Импорт** загружает данные из поддерживаемых форматов и выполняет проверку перед записью в хранилище.
- **Поделиться** создает защищенный пакет для передачи записи другому пользователю.
- QR/key exchange используется для безопасного обмена ключами и данными.

### Настройки безопасности

- В настройках можно выбрать профиль безопасности, параметры автоочистки буфера, автоблокировку, интеграцию с треем, внешний вид и параметры режима паники.
- По умолчанию включены безопасные настройки, а потенциально спорные функции можно включать вручную.


### Режим паники

- Режим паники быстро скрывает чувствительные данные, очищает буфер обмена и блокирует операции с секретами.
- Основная горячая клавиша: `Ctrl+Alt+P`.
- Дополнительные действия режима паники настраиваются в окне настроек.

## Известные ограничения и будущие улучшения

Известные ограничения:

- проект является учебной локальной реализацией менеджера паролей и не проходил независимый внешний аудит безопасности;
- синхронизация между устройствами и облачное хранение не реализованы;
- авто-заполнение в браузере отсутствует, данные копируются через защищенный буфер обмена;
- часть платформенных возможностей зависит от ОС и установленных системных backend-пакетов, особенно для буфера обмена на Linux;
- PyInstaller-сборка может иметь большой размер из-за включения Python runtime, Tkinter, Tcl/Tk и библиотек проекта.

Планируемые улучшения:

- расширение пользовательских профилей и полноценная многопользовательская модель;
- browser extension для авто-заполнения логинов и паролей;
- поддержка TOTP/2FA-кодов;
- зашифрованная синхронизация между устройствами;
- дополнительные темы интерфейса и улучшение доступности;
- расширение набора автоматических security-тестов и сценариев восстановления после сбоев.

## PyCharm + Python 3.14 note

If PyCharm tries to install `pyinstaller==6.0.0` and fails, skip that package. PyInstaller is only needed for building an `.exe`, not for running the app. Run `INSTALL_WINDOWS.bat` to start the app. For `.exe` builds run `INSTALL_BUILD_WINDOWS.bat`, which installs a current PyInstaller version instead of the old `6.0.0` pin.
