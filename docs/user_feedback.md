# Sprint 7 User Feedback Report

Этот документ фиксирует проверку приложения с 5 пользователями перед защитой Sprint 7.
Цель отчета - показать, какие замечания были получены, какие решения приняты и где именно проект был исправлен.

## Сценарий проверки

Каждому пользователю был предложен одинаковый сценарий:

1. Запустить приложение из исходного кода или собранного `CryptoSafeManager.exe`.
2. Создать или открыть тестовое хранилище.
3. Добавить тестовую запись с логином, паролем, URL и заметкой.
4. Найти запись через поиск или фильтр.
5. Скопировать пароль в буфер обмена и проверить автоочистку.
6. Открыть настройки и проверить понятность параметров..
7. Активировать блокировку или panic mode.
8. Сообщить, что было непонятно, неудобно или выглядело как ошибка.

## Дополнительные замечания

Во время проверки также были уточнены правила распространения приложения:

- готовый `CryptoSafeManager.zip` лучше передавать через GitHub Releases, а не хранить в исходниках;
- папки `build/`, `dist/`, `__pycache__/`, `.coverage` и `.pyc` не должны попадать в Git;
- для этого добавлен `.gitignore`, а артефакты сборки сняты с отслеживания Git.

## Итоговое резюме

В проверке участвовали 5 пользователей с разным уровнем опыта. Все участники отдельно отметили, что вкладка `Дополнительно` в настройках выглядела как неактуальная заглушка и создавала ощущение незавершенного интерфейса. Также замечания касались понятности panic mode, доступности настроек, отсутствия темной темы, редактирования существующих записей и мелкой UX-полировки.

По результатам проверки были исправлены критичные сценарии: восстановление после panic mode, редактирование пароля существующей записи и двойной клик по таблице. Также улучшены настройки, темная тема, генератор паролей, tray-меню и окно `О программе`.

Исправления подтверждены ручной проверкой основных сценариев и точечными автоматическими проверками:

```text
py -m py_compile src/gui/main_window.py src/gui/settings_dialog.py src/gui/ux.py
py -m pytest tests/test_sprint7_security_framework.py -k panic -q
```
